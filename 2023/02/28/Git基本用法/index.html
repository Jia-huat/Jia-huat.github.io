Git基本用法

Git的初认识

git创建仓库

  • 每次操作之前都要进行git init
  • 送至暂缓区(修改的文件需要追踪才能提交)
  • -m “修改注释”
  • 一键提交不知道为什么不行

git分支管理

创建分支

  • 第一次提交会有默认分支master
  • git branch -d 文件 (删除分支,切换出才能删除)
  • 切换分支需要先提交修改

分支的细节

  • git log(上一次的提交记录)查看当前分支

1.HEAD→指向当前工作的分支

2.切换分支需要先提交修改

3.分支没有commit到本地时(新的改动)，会保留上一次提交的最初状态。(我理解为，各个分支独立)

分支合并

  • 合并之前，需要提交修改

  • 合并前可以通过git status查看未提交

  • 合并会提示内容差别

远程项目操作

绑定仓库之后直接push就好了
